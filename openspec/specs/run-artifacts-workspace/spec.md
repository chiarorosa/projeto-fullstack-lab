# run-artifacts-workspace Specification

## Purpose
Provide a dedicated artifacts workspace for inspecting persisted run outputs, routing details, and task-level execution outcomes.

## Requirements
### Requirement: Run Artifacts Tab
The frontend SHALL provide a dedicated artifacts tab/workspace for inspecting outputs generated by each run.

#### Scenario: User opens artifacts workspace
- **WHEN** user accesses run artifacts UI for a team
- **THEN** system lists available executions and task-level entries
- **AND** each entry shows status, created time, and selected agent summary

#### Scenario: User inspects task artifact details
- **WHEN** user expands an artifact entry
- **THEN** UI shows task input, routing reason, selected agent, final output or error, and model/provider used

### Requirement: Persisted Routing and Output Metadata
The system SHALL persist routing metadata and outputs in a queryable structure.

#### Scenario: Task completes successfully
- **WHEN** a routed task reaches completion
- **THEN** backend persists task artifact with `completed` status and final output text

#### Scenario: Task fails during routing or execution
- **WHEN** a routed task fails
- **THEN** backend persists task artifact with `failed` status and error message
- **AND** artifact remains visible in run artifacts UI
